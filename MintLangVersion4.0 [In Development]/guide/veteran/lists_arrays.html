Lists and Arrays
Lists are Mint's main data structure for sequential data, declared with square brackets surrounding elements. Arrays offer speed boosts while imposing additional restrictions, declared with the word Array followed by elements in square brackets.

Lists are dynamic arrays, so when their capacity is exceeded they allocate more space to fit additional elements. Lists are also heterogeneous, meaning that a single list can hold elements of differing types, such as a string and an integer in the same list. 
myList = [2, 3, 4, "water", "potassium", 1:2]
nestedList = [[1, 1], [2, 4], [3, 9], [4, 16]]
print myList[0]           // Prints 2
print myList[4]           // Prints "potassium"
print myList[-1]          // Prints 1:2 (the last element)
myList[1] = 300           // Overwrites 3 with 300
print nestedList.pop()    // Prints [4, 16] and removes it from list
print nestedList.length() // Prints 3
newList = []
for each elem of nestedList
    newList.push(elem[0] + elem[1])
end
print newList             // Prints [2, 6, 12]
A list can be indexed backwards with negative integers, where list[-3] simply means list[list.length() - 3].

Arrays are static. They must hold elements of exactly the same type, and items cannot be appended without manually resizing the array. 
/* An array's type is determined by its elements */
intArray = Array[16, 18, 1, 0]
/* An array of type Array(Array(Int)) */
array2D = Array[Array[1, 2], Array[8, 5]]

print intArray[1]        // Prints 18
print intArray[-4]       // Prints 16
print intArray.length()  // Prints 4
for i = 0; i < array2D.length(); i++
    for j = 0; j < array2D[i].length(); j++
        print array2D[i][j]
    end
end

If you want to inspect the type of an array, use the array.type() method. In fact, this method will work on all Mint objects. 
myArray = Array[15.6, 18.7, 9.9, 0.2]
print myArray.type()         // Prints #Array(Real)
intArray = Array[16, 18, 1, 0]
print intArray.type()        // Prints #Array(Int)
print 16.type()              // Prints #Int
print (3:2).type()           // Prints #Ratio
print [1, 2, Array[1, 2]]    // Prints #List
listArray = Array[[8, 9], [1, 2, Array[1, 2]]]
print listArray              // Prints #Array(List)

Notice that making an array of lists is fine. It becomes an array of references to each list, and those lists may still have elements of any type. You may wonder what is actually being printed, since it doesn't look like a string. The obj.type() method returns a type tag marked with a hash symbol (#). Tags will be discussed later.

What is the type of an empty array? The empty array can't infer the type from its elements. Also, what is the best way to create large arrays? 
empty = Array[]
print empty.type()               // Prints #Array
bigArray = Array(#Int, 16000)    // Use parens here, not brackets
print bigArray.length()          // Prints 16000
print bigArray.type()            // Prints #Array(Int)

The empty array, which needs to be resized to be useful, has no type for its elements. If resized to a positive integer, it becomes a rather generic #Array(Object) filled with object placeholders known as null. To make large arrays, call Array as if it were a function (i.e. using parentheses), and give it a type tag followed by the number of elements you want. You can also call Array(#Int, 0) to create an empty typed array, where you may replace #Int with the type of any object you want.
