Operators
Arithmetic, Bitwise, Boolean, and More

Arithmetic Operators
Mint supports the following arithmetic operators. In Mint, double-precision floats are called "real numbers". In the following table, a and b are numeric values, usually an integer, ratio, or real.
OperationSymbolUsage
Addition+a + b
Subtraction-a - b
Multiplication*a * b
Implicit Multiplication(none)a(b)
Division/a / b
Integer DivisionintDivintDiv(a, b)
Exponentiation^a ^ b
Construct Ratio:a:b
Modulo%a % b
Increment++a++
Decrement--a--
The increment and decrement operators must come after the variable (postfix operators). Even though they do return a value, it is highly suggested that you don't use expressions such as x = (y++) * (z--). Instead, expressions like y++ should be on their own line or as part of a for-loop.

Implicit multiplication, which has no symbol, is the act of multiplying by juxtaposing numbers or variables, like in mathematics. Some examples in Mint: 
x = 0.125
y = 2(x)
z = (x+3)y
print z    // Prints 0.78125
What you need to do is place one of the multiplicands in parentheses, and put the other either before or after it.

String Operators
The main operator for strings is concatenation, which uses the + symbol (also used for addition). Mint has two types of strings, regular strings and ribbons. Strings are immutable arrays of characters and ribbons are mutable linked-lists of characters. Both have their pros and cons, and this will be discussed later. A ribbon and a string can be concatenated, in which case the string is converted to a ribbon. Use ribbon.toString() to avoid this.

The only other operator for strings is in. strA in strB returns true if strA is a substring of strB, and false otherwise. Even so, the in keyword can also be used on other types of sequences and collections.

Comparison Operators
Comparison operators return either true or false depending on the values of both their operands.
OperationSymbolUsage
Equal To==a == b
Not Equal To!=a != b
Greater Than>a > b
Less Than<a < b
Greater Than or Equal To>=a >= b
Less Than or Equal To<=a <= b
Note that the single equals sign (=) is used for variable assignment.
== and != work for many objects. They can be used to test equality of numerics, strings, list contents, and more.
The other comparison operators, such as >=, only work on objects that can be properly ordered. This usually means numerics (integers, ratios, reals). However, they will also compare strings by length and compare lists by length. Though you can compare strings with strings and lists with lists, you can't compare a string and a list, unless you only use == and !=.

Bitwise Operators
Bitwise operators treat integers as 32-bit fields and perform operations on those fields.
OperationSymbolUsage
Bitwise And&a & b
Bitwise Or@a @ b
Bitwise Exclusive Or$a $ b
Bitwise Not~~a
Shift Left<<a << b
Shift Arithmetic Right>>a >> b
Shift Logical Rightshlrshlr(a, b)
The choices for bitwise or and bitwise xor look really strange. For bitwise xor, note that ^ is used for exponentiation, so $ is substituted. In the Julia language, which also uses ^ for exponentiation, $ is the bitwise xor operator. For bitwise or, | is used for something we will discuss later, so @ is substituted. You can also use andb, orb, xorb, notb keywords which are equivalent to &, @, $, ~ respectively if you find the keywords easier to read.
