Tables
Mappings of keys to values

Tables are the secondary data structures of Mint. A table is a mapping of keys to values, known as a hashmap in some other languages. A mapping is similar to a phone book, where the phone numbers are the values and the names are the keys.

KeyValue
Joe Smith(220) 555-0888
Bill Richardson(221) 555-2834
Jessica Russell(221) 555-1396

To make a table in Mint, you use comma-separated pairs separated with vertical bars, and surround those with curly braces. 
myTable = {17, 900 | 8, 9 | "honey butter", "gelatin powder"}
phoneNumbers = {"Joe Smith", "(220) 555-0888" | "Bill Richardson", "(221) 555-2834" | "Jessica Russell", "(221) 555-1396"}
To grab an item from a table, use indexing.
print myTable[17]                        //900
print myTable["honey butter"]            //gelatin powder
print phoneNumbers["Bill Richardson"]    //(221) 555-2834
Tables are also ordered collections of pairs, so you can select pairs like this:
print myTable.getPair(0)          //Prints [17, 900]
print myTable.getPair(1)          //Prints [8, 9]
print phoneNumbers.getPair(-1)    //Prints the final pair ["Jessica Russell", "(221) 555-1396"]
Pairs are just lists of exactly 2 items.
To modify a table, use indexing or use setPair. 
myTable[0] = null    //Binds 0 to null
myTable["coffee"] = "16 oz"    //Binds "coffee" to "16 oz"
myTable.setPair("coffee", "16 oz")   //Equivalent to previous line. Does not create a duplicate for "coffee".
print myTable    //Prints {17, 900 | 8, 9 | "honey butter", "gelatin powder" | 0, null | "coffee", "16 oz"}
print myTable["coffee"]    //Prints "16 oz"
print myTable.getPair(-1)    //Prints ["coffee", "16 oz"]
The keyword known as null is an empty object that does nothing. It's just a placeholder.
Vertical bar notation actually wraps comma-separated values in individual lists. It can be used outside of tables to easily create nested lists.
myList = [1, 2, 3 | 1, 4, 9 | 1, 8, 27]
print myList     //Prints [[1, 2, 3], [1, 4, 9], [1, 8, 27]]
